# Посібник: що можна робити в симуляції Clay

> Коротко: **Clay — це не гра, а жива істота, за якою ти спостерігаєш.** Бот сам
> вирішує, чого хоче (новизна, творчість, відпочинок), сам ходить світом, ставить
> блоки й відпочиває. Ти — спостерігач, який дивиться на маленький тераріум. За
> бажанням можеш узяти керування на себе (див. розділ «Ручне керування»).

---

## 1. Два способи запустити

### A. Термінал (headless) — працює будь-де, без графіки

```bash
python3 run_headless.py
python3 run_headless.py --ticks 4000 --map-every 300 --seed 7
```

Нічого встановлювати не треба — лише стандартна бібліотека Python. Цей режим
малює **headless-мапу** (детально нижче) і друкує рішення мозку та смужки потреб.

### B. 3D-вид на Ursina — воксельний світ (локально)

```bash
pip install ursina
python3 game.py
```

Той самий рушій симуляції, але з 3D-картинкою «як у Minecraft»: куби рельєфу,
блакитний бот, помаранчеві поставлені блоки.

---

## 2. Що таке headless-мапа (детально)

Коли запускаєш `run_headless.py`, кожні `--map-every` тіків програма друкує
**мапу зверху** — текстову (ASCII) картинку всього світу. Це **двовимірна
проєкція згори вниз** того самого 3D-світу: висоту ми ігноруємо й дивимося на
сітку клітин рівно зверху.

Малює її функція `draw_map()` у [run_headless.py](run_headless.py).

### Як вона влаштована

- Світ — це сітка `width × depth` клітин (за замовчуванням **32 × 32**, див.
  `World.width` / `World.depth` у [sim/world.py](sim/world.py)).
- **Один символ = одна клітина** світу.
- `x` — це **стовпець** (зліва направо), `z` — це **рядок** (згори вниз).
- Мапа друкується рядок за рядком: зовнішній цикл по `z`, внутрішній по `x`.

### Що означає кожен символ

| символ | значення | звідки береться |
|---|---|---|
| `@` | поточна клітина бота | `sim.body.pos` |
| `#` | клітина, де бот поставив хоча б один блок | `world.placed` |
| `.` | клітина, яку бот **відвідав** (досліджена земля) | `world.visited` |
| ` ` (пробіл) | незаймана, ще не досліджена клітина | — |

**Пріоритет важливий:** спершу перевіряється бот, потім блок, потім відвіданість,
потім порожнеча. Тож якщо бот стоїть на вже відвіданій клітині чи на блоці — ти
все одно бачиш `@` (бот «перекриває» решту).

### Що друкується разом із мапою

Перед мапою йде рядок статистики, наприклад:

```
--- tick 500 | pos (14, 17) | explored 121 | built 3 ---
```

- `tick` — скільки тіків минуло;
- `pos` — координати бота `(x, z)`;
- `explored` — скільки клітин уже відвідано (зростає → росте дослідження світу);
- `built` — скільки всього блоків поставлено.

### Рядки рішень (між мапами)

Окрім мапи, щоразу коли **мозок ухвалює нове рішення**, друкується пара рядків:

```
t=  342  DECIDE  goto -> (9, 21) (restless, seeking new ground)
          needs  novelty [###---------]  creation [########----]  rest [##########--]
```

- `DECIDE` — мозок видав новий **намір** (goto / build / rest / wander / idle) і
  причину в дужках;
- смужки `[####----]` — рівень кожної потреби від 0 до 1 (більше заповнено =
  потреба ситіша). Коли смужка порожніє — бот невдовзі піде її задовольняти.

### Навіщо вона взагалі

Headless-мапа — це найшвидший спосіб **побачити життєвий цикл** без графіки:
дослідження, коли падає новизна; будівництво, коли тягне творити; відпочинок,
коли втомився; дрейф — коли нічого не пече. Той самий рушій працює й під 3D-вікном,
і в юніт-тесті, і на сервері без екрана — мапа просто його «очі» в терміналі.

## 3. Керування у 3D-виді (камера)

У 3D ти за замовчуванням керуєш **лише камерою** (це `EditorCamera` Ursina):

| дія | ефект |
|---|---|
| перетягнути мишею | обертати світ навколо центру |
| коліщатко миші | наблизити / віддалити |
| права/середня кнопка + рух | панорамувати |

Камера тепер стартує **по центру рельєфу** й трохи нахилена вниз, тож світ
одразу видно.

### Кольори у 3D (палітра Minecraft)

Рельєф розфарбований **дискретними блоками за висотою**, як у Minecraft:

| висота | блок | колір |
|---|---|---|
| 0 | вода | синій |
| 1 | пісок | пісочний |
| 2 | трава | зелений |
| 3 | ліс | темно-зелений |
| 4 | камінь | сірий |
| 5+ | сніг | майже білий |

- **блакитний** куб — бот;
- **помаранчеві** куби — поставлені блоки (твої або бота).

> **Чому раніше був чорний екран:** на цьому Mac Ursina отримує контекст
> OpenGL 2.1 / GLSL 1.20 (через Metal), а вбудовані шейдери Ursina написані під
> `#version 130/140` — вони не компілюються, тож усе малювалося чорним силуетом
> (текст HUD працював, бо в нього окремий конвеєр). Виправлено власним
> мінімальним шейдером `#version 120` (`VOXEL_SHADER` у `game.py`), який і
> компілюється тут, і тонує кожну грань за нормаллю (верх найсвітліший) — звідси
> об'ємний воксельний вигляд без жодного джерела світла.

## 4. Ручне керування ботом (нове) — «якщо хочеш сам»

У 3D-виді можеш будь-якої миті **взяти керування на себе**.

| клавіша | дія |
|---|---|
| **M** | перемкнути режим: AUTO (керує мозок) ↔ MANUAL (керуєш ти) |
| **W / A / S / D** або стрілки | крок ботом на сусідню клітину |
| **B** | поставити блок під ботом (годує потребу «творчість») |

Як це працює:

- У режимі **MANUAL** мозок **не рухає** бота — рухаєш його ти. Але **час іде
  далі**: потреби так само спадають. По суті, ти стаєш «політикою» замість мозку.
- Хода на нову клітину **годує новизну**, постановка блоку — **творчість**, точно
  як у автономному режимі.
- Натисни **M** ще раз — і мозок знову бере кермо з поточного стану.

Поточний режим і підказка по клавішах завжди видно у HUD (зверху ліворуч).

## 5. HUD у 3D (що за написи зверху ліворуч)

```
tick 1240   decisions 15   mode: AUTO - brain drives
intent: wander (idling)
novelty 0.86  creation 0.51  rest 0.89
[M] manual on/off    [WASD] move    [B] build
```

- `tick` — лічильник тіків (внутрішній час);
- `decisions` — скільки разів будили мозок;
- `mode` — AUTO чи MANUAL;
- `intent` — що бот робить просто зараз і чому;
- три потреби — рівні `novelty` / `creation` / `rest` (1.0 = ситий);
- останній рядок — підказка по клавішах.

> Правий верхній блок (`fps`, `entities`, `colliders`) — це службова статистика
> Ursina, до симуляції стосунку не має.

## 6. Що відбувається «під капотом» (цикл життя)

```
тіло (швидкі тіки)  ──perception──>  мозок  ──intent──>  тіло
   спад потреб,                       «що далі?»          виконує намір
   рух, виконання наміру                                  сотні тіків сам
```

- **Тіло** живе на швидких тіках само: рухається, спадають потреби, доробляє
  останній намір — усе чистим кодом.
- **Мозок** будять лише коли щось важить: холодний старт, суттєвий намір
  завершено, потреба перетнула поріг під час дрейфу, або спрацював heartbeat.
- **Наміри**, які тіло вміє виконувати:

| намір | що означає |
|---|---|
| `goto` | дійти до клітини (зазвичай — до незвіданої землі) |
| `build` | поставити блок поруч |
| `rest` | відпочити, поки не відновиться |
| `wander` | побродити без мети (тримає бота живим) |
| `idle` | нічого не робити |

## 7. Що можна налаштувати (важелі)

| важіль | де | ефект |
|---|---|---|
| `--seed` | аргумент `run_headless.py` | інший рельєф |
| `--ticks` | аргумент `run_headless.py` | скільки тіків прожити |
| `--map-every` | аргумент `run_headless.py` | як часто друкувати мапу |
| `TICK_INTERVAL` | [game.py](game.py) | швидкість життя у 3D (менше = жвавіше) |
| `GLIDE_SPEED` | [game.py](game.py) | як швидко бот візуально «доїжджає» до клітини |
| `HEARTBEAT` | [sim/body.py](sim/body.py) | максимальна пауза між запитами до мозку |
| `decay` потреб | [sim/needs.py](sim/needs.py) | як швидко наростають потреби |
| `ACT_THRESHOLD` | [sim/brain.py](sim/brain.py) | наскільки терміновою має стати потреба, щоб діяти |

## 8. «Розумний» мозок (LLM / Лілі) — за бажанням

Зараз думає простий мозок-заглушка (`StubBrain`) за кількома правилами. Його
можна без жодних змін у тілі замінити на LLM.

**Уся LLM-конфігурація — у файлі `.env`** (у корені проєкту):

```bash
pip install anthropic
cp .env.example .env     # скопіюй шаблон...
```

Потім відкрий `.env` і встав свій ключ:

| змінна | що це | за замовчуванням |
|---|---|---|
| `ANTHROPIC_API_KEY` | твій ключ Anthropic (порожньо = лишається StubBrain) | — |
| `ANTHROPIC_MODEL` | яка модель «думає» | `claude-haiku-4-5` (швидка й дешева) |
| `LLM_MAX_TOKENS` | ліміт токенів на одне рішення | `200` |

`.env` у `.gitignore` — ключ не потрапить у git. Завантажує його `brains/config.py`
(без зайвих залежностей). Змінні середовища (`export ...`) мають пріоритет над `.env`.

```python
from sim import Simulation, World
from brains.llm_brain_template import LLMBrain
sim = Simulation(world=World(), brain=LLMBrain())   # замість StubBrain
```

Тіло не змінюється — контракт той самий: `request(perception)` / `poll()` /
`busy`. Поки модель «думає» у фоновому потоці, тіло доробляє попередній намір,
тож жодного фризу. Шаблон — у [brains/llm_brain_template.py](brains/llm_brain_template.py).

## 9. Поточні можливості та межі

**Уже можна:**
- спостерігати автономне життя (дослідження / будівництво / відпочинок / дрейф);
- дивитися headless-мапу й смужки потреб у терміналі;
- крутити 3D-камеру навколо світу;
- **самому водити бота** (WASD / B) у 3D і перемикатися AUTO ↔ MANUAL;
- міняти зерно, темп, пороги та підключати LLM-мозок.

**Поки що ні (природні наступні кроки):**
- пошук шляху — жадібний крок, без A* і вартості стрибка на нерівному рельєфі;
- немає збереження стану на диск (після перезапуску світ генерується наново);
- один бот, три загальні потреби (у версії «Лілі» їх буде шість);
- ручне будівництво ставить блок лише під ботом (немає вибору сусідньої клітини).
